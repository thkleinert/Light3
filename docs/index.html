How it works

Organize in Lightroom,
store anywhere

Light3 adds a Publish Service to Lightroom's library panel. Organize photos into collections, hit Publish — done.

01

Organize

Create published collections and collection sets in Lightroom's library panel, exactly like you already do. Drag photos in, arrange them, rate and cull.

02

Publish

Click Publish. Light3 uploads only new or changed photos. Lightroom tracks what's already up-to-date so nothing is re-uploaded unnecessarily.

03

Share

Your photos land in the bucket at a predictable path. Light3 also writes an order.json sidecar so any web gallery can display them in the exact order you arranged them.

Lightroom Classic → light3-sign → S3 bucket
                      (presigns URL locally)   (plain HTTPS PUT, no proxy)

Lightroom's Lua scripting has no crypto, so AWS Signature V4 signing is handled by a small bundled helper binary. Your credentials never leave your machine — the helper generates a presigned URL and the plugin uploads directly.

Features

Everything you need,
nothing you don't

Built for photographers who self-host — not for SaaS platforms that lock in your files.

🗂️

Collection hierarchy

Collection sets map directly to S3 path segments. Weddings/Smith 2026/ in Lightroom becomes galleries/Weddings/Smith_2026/ in your bucket.

🔢

Flexible file naming

Templates with <sequence>, <collection>, <file>, and <uuid> tokens. Use UUID-based names for stable keys that survive reorders.

📋

order.json sidecar

After every publish, Light3 writes a JSON file containing the full ordered list of photo keys — exactly as arranged in Lightroom. Power your gallery without guessing at sort order.

🔒

Credentials stay local

AWS Signature V4 signing happens on your Mac via a bundled helper binary. No proxy, no cloud intermediary, no credentials ever leave your machine.

♻️

Incremental publish

Lightroom tracks which photos are already up-to-date. Only new or edited photos are re-uploaded. Removing a photo from a collection deletes it from the bucket.

🌍

Any S3-compatible storage

One plugin, any endpoint. Switch from R2 to S3 to B2 by changing the endpoint URL — no code changes, no new installs.

order.json — gallery-ready from day one

After every publish, Light3 writes order.json to the collection's S3 prefix. It contains the collection name, the S3 prefix, and the full ordered list of photo keys — exactly as you arranged them by drag-and-drop in Lightroom. Downstream tools (web galleries, slideshow generators) can consume this file without knowing anything about Lightroom's internals.

Installation

Up and running
in five minutes

No Node.js, no Python, no package manager.

1

Download the plugin

Grab the latest .lrplugin.zip from the releases page. The zip includes a pre-built signing helper — no compilation needed.

2

Unzip and add to Lightroom

Unzip to get light3.lrplugin, then either:

Copy it to ~/Library/Application Support/Adobe/Lightroom/Modules/, or open File → Plug-in Manager → Add and point Lightroom at the folder directly.

3

Set up a Publish Service

In the Library module, find Light3 in the Publish Services panel and click Set Up…. Enter your endpoint, bucket, region, and credentials.

4

Create a collection and publish

Drag photos into a published collection under Light3. Click Publish. Done — your photos are in the bucket.

Provider setup

Quick-start settings
for common providers

Copy the settings below into the Light3 publish service configuration.

In the Cloudflare dashboard go to R2 → your bucket → Settings → S3 Auth and create an API token with Object Read & Write permissions. Your Account ID is on the R2 overview page.

Endpoint URLhttps://<account-id>.r2.cloudflarestorage.com
BucketYour R2 bucket name
Regionauto
Access Key IDR2 token Access Key ID
Secret Access KeyR2 token Secret Access Key

Create an IAM user with an inline policy granting s3:PutObject and s3:DeleteObject on your bucket, then generate access keys for that user. 

{
  "Effect": "Allow",
  "Action": ["s3:PutObject", "s3:DeleteObject"],
  "Resource": "arn:aws:s3:::your-bucket-name/*"
}
FieldValue
Endpoint URLhttps://s3.<region>.amazonaws.com
BucketYour bucket name
Regionus-east-1 (or your bucket's region)
Access Key IDIAM access key
Secret Access KeyIAM secret key

In Backblaze B2, create an Application Key with read and write access to your bucket. The S3-compatible endpoint is shown in the bucket details page.
